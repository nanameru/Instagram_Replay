# Instagram API トークンタイプ解説

## Instagram APIで使用できる主なトークンタイプ

### 1. ユーザーアクセストークン
- **説明**: 個人のInstagramアカウントに関連付けられたトークン
- **権限**: 基本的なプロフィール情報の取得が可能
- **制限**: DMへのアクセス権限（`read_mailbox`）は通常含まれない
- **有効期限**: 短期（約1〜2時間）または長期（約60日）

### 2. ページアクセストークン
- **説明**: Facebookページに関連付けられたトークン
- **権限**: ページに関連するInstagramビジネスアカウントのデータにアクセス可能
- **制限**: ビジネスアカウントに関連付けられている必要がある
- **有効期限**: 長期（約60日）、更新可能

### 3. インスタグラムビジネスアクセストークン
- **説明**: Instagramビジネスアカウントに特化したトークン
- **権限**: メディア、インサイト、コメントなどのビジネス機能にアクセス可能
- **制限**: ビジネスアカウントが必要
- **有効期限**: 長期（約60日）、更新可能

### 4. アプリトークン
- **説明**: アプリケーション自体に関連付けられたトークン
- **権限**: アプリケーションレベルの操作に使用
- **制限**: ユーザーデータへのアクセスには別途ユーザートークンが必要
- **有効期限**: 無期限（アプリが有効な限り）

## DMアクセスに必要なトークンと権限

### 必要なトークンタイプ
DMにアクセスするには、**ページアクセストークン**が最適です。これはFacebookページに関連付けられたInstagramビジネスアカウントのデータにアクセスするためのトークンです。

### 必要な権限
1. **`read_mailbox`**: DMの読み取りに必要
2. **`manage_messages`**: メッセージの送信に必要

### 権限取得のプロセス
1. Facebookデベロッパーアカウントを作成
2. アプリを作成し、Instagramプラットフォームを追加
3. 必要な権限（`read_mailbox`、`manage_messages`）をリクエスト
4. Facebookのアプリレビュープロセスを通過（ビジネス検証が必要な場合あり）
5. 承認後、適切なトークンを生成

## 推奨事項

1. **ビジネスアカウントの使用**: InstagramビジネスアカウントとFacebookページを連携させる
2. **適切な権限のリクエスト**: アプリレビューを通じて`read_mailbox`と`manage_messages`権限を取得
3. **トークン管理**: 長期トークンを使用し、定期的に更新する仕組みを実装
4. **フォールバックメカニズム**: 権限が制限されている場合のモックデータ使用などの代替手段を用意
